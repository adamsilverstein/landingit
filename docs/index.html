What it does

Every PR and issue across every repo. In one list.

GitHub's feed is firehose. Your inbox is a graveyard. LandinGit is the sober middle — a single dense list where things that need you rise to the top, and everything else stays quiet.

01Multi-repo

Every repo, one list.

Hook up as many repositories as you own or watch. One sortable, filterable, keyboard-navigable inbox.

02Keyboard

j/k your day.

Every action has a shortcut. Move, filter, open, triage — your hands never touch the mouse.

03Labels

Faithful color.

Your labels render with the exact colors you set on GitHub — so bugs look like bugs and features look like features.

04Filters

Built for triage.

Filter by owner, type, state, label, milestone, or your own replies. Save filter sets and switch with a keystroke.

05Menu bar

Quietly watching.

A monochrome bird in your menu bar with live counts. Opens the full list with ⌘⇧G — land, then close.

06iOS

On the go.

The same triage flow in your pocket. Swipe to open, long-press to mark, pull to refresh. Still keyboard-first on iPad.

Keyboard

Learn ten keys. Never look back.

The home row is the whole UI. Mouse support is a courtesy, not a requirement — and ⌘ K opens the rest when your muscle memory hasn't caught up yet.

FAQ

Questions, answered.

Does it work with private repositories?
Yes. Sign in with GitHub and grant access to any combination of personal, org, or private repos. You control the scope at any time.
Does LandinGit store my GitHub data?
No. Your token lives in the macOS Keychain, requests go directly from your device to GitHub, and list state is cached locally. We do not have a server with your data on it.
Will you support GitLab or Bitbucket?
Not in 1.0. GitHub is the only forge we support today. Gitea and GitLab are on the roadmap if enough people ask.
Is there a web version?
Yes — a web build runs in any modern browser. Native macOS and iOS get the menu bar and notification integrations the web build can't do.
How does the menu bar counter work?
It polls your repos on an interval you control (default 2 minutes) and shows the count of items that have changed since you last opened the app. Click it to open the list; the count resets when you catch up.
What's the difference between "draft" and "open"?
Those are GitHub's own PR states, surfaced faithfully. LandinGit never reinterprets a state — it just renders them with a consistent pill color so you can scan hundreds of rows at a glance.
